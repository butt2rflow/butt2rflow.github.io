 > 몬테카를로 방법(Monte Carlo method)은난수를 이용하여 함수의 값을확률적으로 계산하는알고리즘을 부르는 용어이다.수학이나물리학등에 자주 사용되며, 계산하려는 값이닫힌 형식으로 표현되지 않거나 복잡한 경우에근사적으로 계산할 때 사용된다.스타니스와프 울람이모나코의 유명한 도박의 도시몬테카를로의 이름을 본따 명명하였다.1930년대 엔리코 페르미가중성자의 특성을 연구하기 위해 이 방법을 사용한 것으로 유명하다.맨해튼 계획의 시뮬레이션이나수소폭탄의 개발에서도 핵심적인 역할을 담당하였다.알고리즘의 반복과 큰 수의 계산이 관련되기 때문에 몬테카를로는 다양한컴퓨터 모의 실험기술을 사용하여컴퓨터로 계산하는 것이 적합하다.Monte Carlo 방법(또는 Monte Carlo 실험)은 수학적인 결과를 얻기 위해 반복적으로 무작위 샘플링의 방법을 이용하는 넓은 범위의 컴퓨터 알고리즘이다. 이 알고리즘의 본질적인 생각은 결정론적일 수도 있는 문제를 해결하기 위해 무작위성을 이용하는 것이다. 이 방법은 보통 물리나 수학 문제를 해결하는 데 쓰이며, 다른 방향으로의 접근이 불가능할 때 가장 유용하다. Monte Carlo 방법은 주로 최적화, 수치적 통합, 확률 분포로부터의 도출 등에서 주로 사용된다.

  몬테 카를로 시뮬레이션은 계산 하려는 값이 닫힌 형식으로 표현 되지 않거나 복잡한 경우에 근사적으로 계산 할 때 사용된다. 그리고 수소 폭탄의 개발에서도 핵심적인 역할을 담당 하였다고 해서 우리가 쫄(?) 이유는 없습니다.

    금융 공학이나 증권사에서도 실전으로 몬테 카를로 시뮬레이션으로 주식 시장에서의 투자 모델을 백테스팅(backtesting) 한다는 이야기는 제법 자주 들을 수 있습니다. 누군가 몬테 카를로 시뮬레이션으로 백테스팅 한다고 하면 괜히 멋져 보입니다.

    어려워 보이는 주제를 이해 하려고 할 때에는 튜토리얼을 찾아보고 "Hello World" 프로그램을 직접 작성해 보는 것이 제법 효과가 좋습니다. 그래서 이 번에 몬테 카를로 시뮬레이션 용 "Hello World" 프로그램을 구글 시트에서 작성해 보고자 합니다.

 > '2020년의 QQQ 주가 데이터'를 사용하여 '투자 금액 $1,000'을 '50일 동안' 투자 하는 경우를 '몬테 카를로스 시뮬레이션'을 '1,000 번 반복' 하여  백테스팅 해보자.

  학교를 졸업하고 확률, 통계 함수를 사용할 일은 별로 없습니다. 전에 배웠든 안 배웠든 상관 없이 기억이 날 리가 없습니다. 간단한 기본 개념을 하나 하나씩 먼저 짚어 볼 필요가 있을 것 같습니다.

    *RAND()***함수**

    *RAND()* 함수는 동등 확률(equal probability)로 매 번 호출 할 때 마다 0에서 1사이의 임의의 실수를 돌려 주는 함수 입니다.  동등 확률로 돌려 준다는 의미는 즉, 균등 분포(Uniform Distribution) 함수라는 의미 입니다.

  *RAND()* 함수를 10번 실행 하고 그 결과를 히스토그램으로 그려 보면 균등 분포가 아닌가 싶습니다. 베르누이 '대수의 법칙(Law of large numbers)'에 따르면 표본 값의 숫자가 대단히 클 때 통계적 확률과 관찰 되는 확률이 수렴 한다고 했으니 표본 값의 숫자를 늘려 봅니다.

  100회, 1,000회 에서 10,000회 까지 반복 실행 수를 증가하니 히스토그램에서 점점 균등 분포를 보이기 시작 하는 것을 확인할 수 있습니다.

    **확률 분포(Probability Distribution) 함수**

    매 년 크리스마스 카드를 판매 했던 수량의 지난 10년 동안의 기록에 따르면 다음과 같은 확률이 나왔습니다.  10,000 장 판매 10%, 20,000 장 판매 35%, 40,000 장 판매 30% 그리고 60,000 장 판매 25%

  위와 같은 확률로 표에 해당 하는 판매 수량 중 하나의 숫자가 무작위로 나오는 함수를 만들고 싶다면 확률표의 확률 길이에 해당 하는 막대기를 옆으로 나란히 길게 나열하면 됩니다.

  빨간색 막대기의 길이는 0.1(10%), 노란색 막대기의 길이는 0.35(35%), 녹색 막대기의 길이는 0.3(30%), 파란색 막대기의 길이는 0.25(25%) 입니다. 따라서 막대기의 전체 길이는 1 (0.1+0.35+0.3+0.25)이 됩니다.

![RAND()와 확률분포 — 0~1 구간을 확률 비율로 나누기](../assets/diagrams/mc_probability_bar.png)

    *RAND()* 함수는 호출할 때 마다 동등한 확률로 0 에서 1 사이의 임의의 실수를 돌려 준다고 했습니다. *RAND()* 함수가 돌려주는 임의의 실수를 막대기에서 해당 하는 숫자에 매핑 해봅니다.

    *RAND()* -> 0.05 <-> 빨간색

  *RAND()* -> 0.25 <-> 노란색

  *RAND()* -> 0.50 <-> 녹색

  *RAND()* -> 0.80 <-> 파란색

    *RAND()*를 호출할 때 마다 동등한 확률로 0 에서 1 사이의 숫자를 돌려 받고 이 숫자를 막대기에 매핑 한다면 다음과 같은 확률을 얻을 수 있습니다.

    빨간색에 해당 하는 숫자를 얻을 확률 10% = 10,000 장 판매 할 확률

  노란색에 해당 하는 숫자를 얻을 확률 35% = 20,000 장 판매 할 확률

  녹색에 해당 하는 숫자를 얻을 확률 30% = 40,000 장 판매 할 확률

  파란색에 해당 하는 숫자를 얻을 확률 25% = 60,000 장 판매 할 확률

    위의 막대기에서 경계에 해당 하는 숫자들은 다음과 같습니다.

    빨간색과 노란색의 경계 = 0.1 (0 + 0.1)

  노란색과 녹색의 경계 = 0.45 (0 + 0.1 + 0.35)

  녹색과 파란색의 경계 = 0.75 (0 + 0.1 + 0.35 + 0.30)

    위 숫자들을 사용 하여 다음과 같은 표를 만들고 '누적 분포 역함수(Inverse Cumulative Distribution) 표'라고 부르겠습니다. 이 표는 위에서 만든 전체 막대기에 해당하는 표가 됩니다.

  *RAND()* 를 호출 하고 돌려 받은 0 에서 1 사이인 임의의 실수를 막대기에 매핑하는 것을 *VLOOKUP()* 함수로 구현 합니다. *VLOOKUP()* 함수는 두 번째 인자로 주어지는 테이블에서 첫 번째 인자로 주어지는 숫자를 매핑하는 기능을 합니다. 전체 막대기에서 *RAND()*가 돌려 주는 숫자에 해당하는 색깔(즉, 카드 판매 수량)을 매핑 해서 돌려 주는 역할을 하게 됩니다.

 ﻿=VLOOKUP(RAND(), '누적 분포 역함수 표', 2)  제대로 동작 하는 지 확인 하기 위해서 1,000 번 반복한 결과를 다음과 같이 히스토그램에 그려 봅니다.

  1000 번 중에서 100번 조금 못 미치게 10,000 장(빨간색), 350번 조금 넘게 20,000 장(노란색), 300번 조금 못 미치게 40,000 장(녹색), 250번 조금 넘게 60,000(파란색)의 결과를 보여 줍니다.

    RAND() 를 실행 할 때 마다 동등한 확률로 돌려 주는 0에서 1사이의 임의의 실수를 사용 하여 아래와 같은 테이블의 확률에 수렴 하는 결과를 돌려 주는 새로운 함수를 작성 했습니다.

  이 때 사용 한 **VLOOKUP(RAND(), '누적 분포 역함수 표', 2)**가 '**확률 분포(Probability Distribution) 함수**'가 됩니다.

    '확률 분포 함수'를 구하고 나면 몬테 카를로 시뮬레이션은 다 한 것이나 다름 없습니다. 다음과 같은 카드 판매 가격/제작 비용/처분 비용이 든다고 합니다.

  만약 올 해 40,000 장을 제작 하기로 한다면 다음과 같은 손익 계산서를 예상해 볼 수 있습니다. 바로 앞에서 구현 한 '확률 분포 함수'를 사용 합니다.

  공급(Produced) = 40,000

  수요(Demand) = VLOOKUP('RAND(),'누적 분포 역함수 표',2)

  매출(Revenue) = '판매 가격' * MIN('공급' , '수요')

  총 제작 비용(Total var cost) ='제작 비용' * '공급'

  총 처분 비용(Total Disposing cost) = '처분 비용' * IF('공급' > '수요', '공급' - '수요', 0)

  손익(Profit) = '매출' - '총 제작 비용' - '총 처분 비용'

    매번 *RAND()* 함수를 호출 할 때마다 '확률 분포 함수'에 따라서 수요(demand)가 변경 되고 변경 된 수요에 따라서 손익(profit)이 자동으로 계산 됩니다.

    이 시점 부터는 몬테 카를로 시뮬레이션을 하기 위해서 여러 가지 방법이 존재 합니다. 엑셀의 경우 'What If Analysis' 도구를 사용하여 시뮬레이션을 할 수 있습니다.

  여기서는 약간의 노가다를 통하여 외부 도구의 도움 없이 구글 시트의 기본 기능으로만 몬테 카를로 시뮬레이션을 합니다. 이를 위해서 손익(Profit)을 한 줄 계산식으로 변경 합니다.

    손익(Profit)='판매 가격'*MIN('공급',VLOOKUP('임의의 실수','누적 분포 역함수 표',2))-'제작 비용'*'공급'-'처리 비용'*IF('공급'>VLOOKUP(''임의의 실수','누적 분포 역함수 표',2),'공급' - VLOOKUP('임의의 실수','누적 분포 역함수 표',2),0)

    앞서 몇 단계로 계산 했던 손익(Profit)을 한 줄 계산식으로 변경 하고 나서 문제가 생깁니다. '임의의 실수' 부분에 *RAND()*를 넣어 주면 위의 한 줄 계산식에서 RAND()를 세 차례 호출 하고 각각의 *RAND()* 가 돌려주는 임의의 실수 값은 매 번 변경 됩니다. 즉, 세 차례 *RAND()*를 호출 할 때 마다 수요(demand)가 다른 값으로 계산 된다는 의미가 됩니다.

    이 문제를 해결 하기 위해서 스크립트 편집기로 커스텀 함수를 추가 합니다. 커스텀 함수는*NOW()*, *RAND()* 와 같은 함수의 결과 값을 입력 인자로 받을 수 없다는 제약이 있기 때문에 *VLOOKUP()* 에 해당하는 기능도 직접 구현 해야 합니다.

 function GetProfit(produced, demandTable, unit_sale, unit_cost, unit_disposal)
{
  var demand = getVLOOKUP(Math.random(), demandTable, 2);
  var revenue = unit_sale * Math.min(produced, demand);
  var production_cost = unit_cost * produced;
  var dispsal_cost = unit_disposal * (produced > demand ? produced - demand : 0);

  return revenue - production_cost - dispsal_cost;
}

function getVLOOKUP(search_key, range, index)
{
  var found = 0;
  for(var i in range)
  {
    if(search_key < range[i][0]) break;
    found = i;
  }
  return range[found][index-1];

    공급(produced)이 10,000 장 / 20,000 장 / 40,000 장 / 60,000 장일 때를 500번 반복 시뮬레이션할 수 있게 위와 같은 표를 만들고 각각의 경우 첫 번째 행(row)에 위에서 작성한 한 줄 손익(profit) 계산식을 입력 합니다.

  한 줄 계산식을 입력한 첫 번째 행을 모두 선택 하고 오른 쪽 아래 코너의 파란색 사각점을 더블 클릭 하면 각각 경우의 한 줄 계산식이 500 회 씩 확장 하면서 500번의 몬테 카를로 시뮬레이션을 수행 합니다.

  500 번 몬테 카를로 시뮬레이션 한 각각의 결과에 대한 평균(mean)과 표준편차(standard deviation)에 의하면 60,000 장을 공급 하는 것은 20,000 장이나 40,000 을 공급하는 것에 비해 유리한 점이 전혀 없고 안정적인 수입을 원하는 경우에도 10,000장을 공급하는 것 보다는 20,000 을 공급하는 것이 유리하다는 사실을 알 수 있습니다.

  공급을 10,000 장, 20,000 장, 40,000 장, 60,000 장 공급 하는 각각의 경우의 500 회 시뮬레이션 결과를 시각 적으로 확인 하기 위해서 히스토그램에 입력해 보면서 첫 번째 몬테 카를로 시뮬레이션을 마무리 해 봅니다.

    **'확률 분포(Probability Distribution) 함수', '누적 분포(Cumulative Distribution) 함수', '역 누적 분포(Inverse Cumulative Distribution) 함수'**

  다양한 종류의 '확률 분포 함수'가 존재 하지만 대게의 사람 들은 대부분 들어 본 적 조차 없을 것입니다. 하지만 종 모양의 '정규 분포(Normal Distribution) 함수'는 모두 한 번 즈음은 들어 본 적이 있을 것입니다.

    사람들의 평균(mean) IQ가 100 이고 표준 편차(standard deviation)가 15 인 경우의 '정규 분포(Normal Distribution) 함수'를 살펴 봅니다.

  (1) 어떤 사람이 IQ가 120 이상 일 확률은? 9.12%

  '정규 분포 함수'의 전체 영역의 합은 1 (100%) 일 때 IQ 가 120 이상 일 확률에 해당 하는 것은 빨간색 빗금친 영역의 넓이가 됩니다. 이런 경우 파란색 빗금친 영역의 넓이를 구하고 그 값을 1 에서 뺀 값을 구하게 됩니다. 구글 시트에서는 다음과 같은 계산식으로 구할 수 있습니다. *NORM.DIST()* 함수의 4번째 인자가 'true' 인 경우 '누적(cumulative) 함수'를 사용 한다는 의미가 됩니다.

 =1-NORM.DIST(120,100,15,true)  (2) 하위 40%의 사람들의 IQ는 얼마 보다 낮을까요? 96.2

  이 경우에는 구글 시트에서 다음과 같은 계산식으로 구할 수 있습니다.

 =NORM.INV(0.4,100,15)  여기서  *NORM.DIST(,,,true)*는 '정규 분포 함수'의 '누적 분포(Cumulative Distribution) 함수'를 의미 하고 *NORM.INV()* 는 '역 누적 분포(Inverse Cumulative Distribution) 함수'를 의미 합니다. '누적(Cumulative) 함수'의 개념은 앞서 확률에 따라 다른 길이의 막대기를 눕혀서 길게 연결 하는 개념과 똑같습니다. '정균 분포 함수'를 단순화 시킨 '확률 분포 표'를 만들어 봅니다.

  위의 '확률 분포 표'와 막대기와 *VLOOKUP()* 함수를 사용하면 앞의 크리스마스 카드 예제에서와 같은 방식으로 '확률 분포 함수'를 사용할 수 있습니다.

    이산(discrete)적인 '확률 분포 표'와 막대기의 개념을 촘촘하게 무한대로 쪼개서 연속적인 개념으로 만든 것이 다음과 같은 '누적 분포(Cumulative Distribution) 함수'가 됩니다.

  (3) 어떤 사람의 IQ가 96.2 보다 낮을 확률은? 40%

 =NORM.DIST(96.2,100,15,true)  앞서 언급 했던 *NORM.DIST()*의 네 번째 인자가 'true' 이면 '누적 함수'를 사용한다는 의미는 바로 '누적 분포 함수' 상에서 동작 한다는 의미 였습니다. *NORM.INV()* 함수는 '누적 분포 함수'의 '역(Inverse) 함수'가 됩니다. IQ 를 입력으로 확률을 구할 때는 '누적 분포 함수' *NORM.DIST( , , ,true)*를 사용 하고 반대로 확률을 입력으로 IQ를 구할 때는 '누적 분포 역함수'인 *NORM.INV()*를 사용 합니다.

    여기서 중요한 키 포인트 입니다. ('역 누적 분포 함수' 와 '누적 분포 역함수'는 같은 의미라 생각 합니다.)

 > 확률을 입력으로 받는 '역 누적 분포(Inverse Cumulative Distribution) 함수'의 입력으로RAND()를 주면 '역 누적 분포 함수'의 '확률 분포(Probability Distribution) 함수'를 따르는 '임의의 변수(Random Variable)'를 얻을 수가 있습니다.

  '정규 분포(Normal Distribution) 함수'의 '역 누적 분포 함수' 는 *NORMINV()*입니다. '정규 로그 분포(Log Normal Distribution) 함수'의 역 누적 분포 함수'는 *LOGINV()* 입니다.

    '정규 분포'를 따르는 임의의 변수는 다음과 같이 생성 할 수 있습니다.

 =NORMINV(RAND(), mean, stdev)  '정규 로그 분포'를 따르는 임의의 변수는 다음과 같이 생성 할 수 있습니다. 이 경우에는 평균은 *LN(mean)*, 표준 편차는 *LN(stdev)* 로 입력 해야 합니다.

 =LOGINV(RAND(), mean_of_ln, stdev_of_ln)

**주의:** LOGINV()의 mean, stdev 파라미터는 로그정규분포 자체의 평균/표준편차가 아니라, **기저 정규분포(ln(X))의 평균과 표준편차**입니다. 가격 데이터를 사용할 경우 먼저 LN()으로 변환한 값의 평균과 표준편차를 구해서 입력해야 합니다.

'정규 분포' 임의의 변수와 '정규 로그 분포' 임의의 변수를 각각 1000번씩 생성한 후에 히스토그램에서 시각적으로 확인해봅니다.

    **2020년 한 해 동안 QQQ 수익률의 '확률 분포 함수'**

    QQQ의 2020년 일 년 동안의 하루 단위 종가를 다음과 같이 가져 옵니다.

 =GOOGLEFINANCE("QQQ","CLOSE","2020/1/1","2020/12/31")  하루 단위의 종가 기준 수익률(Rate or Return)을 다음과 같이 계산 합니다.

 =COUNT(C3:C1000)  계산 한 수익률의 데이터 총 갯수는 251개 입니다. 2020년 일 년 동안 252 거래일이 있었고 첫 날 수익률은 계산 하지 않았기 때문에 251 개가 됩니다.

    위에서 계산 한 251 개 수익률의 히스토그램은 다음과 같습니다.

  수익률의 평균(mean)과 표준편차(stdev)를 계산 하여 '정규 분포 함수'를 구하면 '누적 분포 역함수'를 사용 하여 QQQ의 2020년 수익률의 '정규 분포'를 따르는 임의의 변수를 생성 하는 것이 가능 합니다.

  '정규 분포 함수(Normal Distribution Function)'

 =NORM.DIST( ,MEAN_ROR_QQQ2020,STDEV_ROR_QQQ2020,false)  '정규 누적 분포 함수(Normal Cumulative Distribution Function)'

 =NORM.DIST( ,MEAN_ROR_QQQ2020,STDEV_ROR_QQQ2020,true)  '정규 누적 분포 역함수(Inverse Normal Cumulative Distribution Function)'

 =NORMINV( ,MEAN_ROR_QQQ2020,STDEV_ROR_QQQ2020)  따라서 QQQ의 2020년 수익률의 '분포 함수'를 따르는 임의의 수익률은 '정규 누적 분포 역함수'에 *RAND()* 를 사용하면 됩니다.

 =NORMINV(RAND() ,MEAN_ROR_QQQ2020,STDEV_ROR_QQQ2020)  1000번을 반복 하여 임의로 생성한 1000 개의 수익률의 히스토그램은 다음과 같습니다.

  위에서 실재로 계산 한 251개 수익률의 히스토그램과 비교해 봅니다.

![QQQ 2020년 일간 수익률 분포 vs 정규분포](../assets/diagrams/mc_qqq_histogram.png)

  전체 적인 모양은 종 모양의 정규 분포 함수로 서로 유사 하지만 실재로 계산 한 251개 수익률에서 보이는 '**팻 테일(fat tail)**'이 *NORMINV()* 함수를 사용 한 수학 모델에서는 사라진 것을 볼 수 있습니다.

![정규분포 vs 실제 분포 — Fat Tail이 보인다](../assets/diagrams/mc_fat_tail.png)

    금융권에서 실전에서는 몬테 카를로 시뮬레이션으로 백테스팅 할 때 '경험적인 분포(Emperical Probability Distribution) 함수'를 사용 하여 팻 테일 상황을 비롯한 추가적인 파라미터를 입력 하여 사용 한다고 합니다.

    그 들이 어떻게 분포 함수를 구현 하여 사용 하는 지 알 수 는 없지만 이 글을 준비하면서 아주 쌈박한 방법을 찾아 냈습니다. 앞에서 설명 하면서 사용 했던 막대기를 눕힌 후 길게 연결 한 후*RAND()*를 매번 호출 할 때 마다 생성 하는 0 에서 1 사이의 숫자를 매핑하는 방법을 이용 합니다.

    251개의 실재로 계산한 수익률을 길게 연결 한 후 1 에서 251 사이의 임의의 정수를 생성 하여 해당 정수에 해당 하는 수익률을 사용 합니다. *RANDBETWEEN()*은  특정 되어진 범위 내 에서 '임의의 정수'를 돌려 주는 함수 입니다. 편의 상 251개의 수익률의 배열을 구글 시트의 '이름 구간(Named range)' 기능을 사용 하여 'ROR_QQQ2020' 이라고 정의 합니다.

  따라서 QQQ의 2020년 수익률의 '분포 함수'를 따르는 새로운 함수는 다음과 같이 됩니다.

 =INDEX(ROR_QQQ2020,RANDBETWEEN(1,251))  새로운 함수를 1000번을 반복 하여 임의로 생성한 1000 개의 수익률의 히스토그램은 다음과 같습니다. '팻 테일'이 재현 되는 것에 주목 합니다.

  **몬테 카를로 시뮬레이션 백테스팅**

    앞에서 사용 할 '확률 분포 함수'를 구하고 나면 몬테 카를로 시뮬레이션은 끝난 것이나 다름 없다고 했습니다.

 > '2020년의 QQQ 주가 데이터'를 사용하여 '투자 금액 $1,000'을 '50일 동안' 투자 하는 경우를 '몬테 카를로스 시뮬레이션'을 '1,000 번 반복' 하여  백테스팅 해보자.

  50일 동안 1,000 번 반복하는 몬테 카를로 시뮬레이션을 하기 위해서는 50 열 x 100 행 의 셀을 사용 합니다.

  50 열은 사용하기 위해서 AX 열 까지 수동으로 행을 추가 해준 후 각각 열의 첫번 째 셀에 앞에서 만든 함수 계산식을 넣어 줍니다.

 =INDEX(ROR_QQQ2020,RANDBETWEEN(1,251))  계산식을 입력 한 첫 번째 행을 모두 선택한 후 오른 쪽 하단의 사각 점을 더블 클릭하면 아래로 1000 행 까지 계산식이 자동으로 입력 됩니다.(더블 클릭 자동 입력이 작동 하지 않으면 사각 점을 클릭 하여 홀드 한 후 1000 행 까지 드래그 할 수도 있습니다.)

  **​**

  **몬테 카를로 시뮬레이션 백테스팅이 막 끝났습니다. ?!?!?**

    편의 상 위에서 계산한 몬테 카를로 시뮬레이션 50 x 100 결과 테이블을 'MC_QQQ2020_TABLE_50' 이라 정의 합니다.

  몬테 카를로 시뮬레이션의 결과물을 쉽게 확인 할 수 있도록 몇 가지 테이블과 결과 차트를 추가하기 위해 결과물 테이블을 재 가공 합니다.

  시작 할 때 투자 금액은 모두 $1,000로 시작하며 몬테 카를로 시뮬레이션으로 생성 한 임의로 생성한 수익률을 사용하여 첫 째날의 수익을 계산 합니다.

 =B2*(1+INDEX(MC_QQQ2020_TABLE_50,$A2,C$1))  위의 계산식을 나머지 셀 들에도 모두 동일 하게 적용 합니다.

## 파이썬 버전

위의 구글 시트 시뮬레이션을 파이썬으로 구현하면 구글 시트의 제약(차트 99개 시리즈 한계, 느린 속도) 없이 실행할 수 있습니다. 핵심 코드는 다음과 같습니다:

```python
import numpy as np
import yfinance as yf

# QQQ 2020년 데이터 다운로드
qqq = yf.download("QQQ", start="2020-01-01", end="2020-12-31")
closes = qqq["Close"].values.flatten()
log_returns = np.diff(np.log(closes))  # 251개 일간 로그 수익률

mu, sigma = log_returns.mean(), log_returns.std()

# 방법 1: 정규분포 기반
random_returns = np.random.normal(mu, sigma, size=(1000, 50))

# 방법 2: 경험적 분포 (부트스트랩) — Fat Tail 보존
random_returns = np.random.choice(log_returns, size=(1000, 50), replace=True)

# 시뮬레이션 경로 계산
paths = 1000 * np.exp(np.cumsum(random_returns, axis=1))
```

## 마무리

몬테카를로 시뮬레이션의 핵심을 정리하면:

1. **RAND()로 균등분포 난수 생성** → 확률분포 함수에 매핑
2. **역누적분포함수(Inverse CDF)** + RAND() = 원하는 분포를 따르는 임의의 변수
3. **정규분포 기반**: NORMINV(RAND(), mean, stdev) — 깔끔하지만 Fat Tail이 사라짐
4. **경험적 분포(부트스트랩)**: INDEX(실제데이터, RANDBETWEEN()) — Fat Tail 보존
5. 충분히 많이 반복하면 대수의 법칙에 의해 통계적 확률에 수렴

금융 실전에서는 경험적 분포 + 추가 파라미터(변동성 클러스터링, 점프 모델 등)를 결합하여 더 정교한 시뮬레이션을 수행합니다. 하지만 원리는 여기서 다룬 "Hello World"와 정확히 같습니다.
